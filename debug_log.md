## 📅 2026-02-13 修正与成功

- **问题**: 刷写成功但 LED 不亮。
- **排查**: 用户指出板载 LED 实际连接在 **PB0** (而非 BluePill 常见的 PC13)。
- **修复**: 修改 `main.c` 中的宏定义，并开启 `GPIOB` 时钟。
- **结果**: **LED 点亮 (成功)**。
- **遗留现象**: 短按复位键 LED 不亮，长按复位键 LED 亮。
    - **分析**: 这是典型的 **ASP 电路干扰**。USB 串口芯片的 DTR/RTS 信号在默认状态下可能拉高了 BOOT0。
    - **短按**: 采样到 BOOT0=1 -> 进入 ISP 下载模式 (程序不运行)。
    - **长按**: 电容充放电或信号抖动导致采样到 BOOT0=0 -> 进入用户程序模式。

## 📅 2026-02-14 案例02: LED 呼吸灯

- **问题**: 代码烧录后 LED 不闪也不亮。
- **排查**: `HAL_Delay` 和 `HAL_GetTick` 依赖 SysTick 中断，但未实现 `SysTick_Handler`。
- **解决**: 在 `main.c` 中添加 `SysTick_Handler` 调用 `HAL_IncTick()`。
- **状态**: ✅ 已解决。PB0 闪烁，PB1 呼吸。

## 📅 2026-02-14 案例03: 按键控制 LED

- **问题 1**: 按键逻辑反向 (按下灭，松开亮)。
- **分析**: 之前假设 LED 是低电平点亮 (Active Low)，但实际测试发现板载 LED 是 **高电平点亮 (Active High)**。
- **解决**: 
    - 初始化时 `GPIO_PIN_RESET` (灭)。
    - 点亮逻辑改为 `WritePin(..., GPIO_PIN_SET)`。
- **经验**: 开发板的 LED 极性不尽相同，编写代码前最好查阅原理图或通过简单高低电平测试确认。
- **状态**: ✅ 已解决。PA0 完美翻转，PA1 完美点动。

## 📅 2026-02-14 案例04: 蜂鸣器音乐

- **问题 1**: 初始刷写后蜂鸣器无声。
- **排查**: 编写“盲测程序”同时翻转 PB5/PB12/PB8/PA8。确认 PB5 有声，证明硬件连接无误。
- **分析**: PB5 是特殊引脚 (JTAG/TRACESWO)。默认被调试接口占用。
- **解决**:
    1.  开启 AFIO 时钟。
    2.  调用 `__HAL_AFIO_REMAP_SWJ_NOJTAG()` 禁用 JTAG，释放 PB5。
    3.  配置 TIM3 Partial Remap。
- **问题 2**: 播放音乐时偶尔无声或卡顿。
- **分析**: 动态修改 PWM 频率 (`ARR` 寄存器) 后，默认要等到下一个计数周期溢出才生效。如果当前 ARR 很大，更新会延迟。
- **解决**: 在修改 ARR 后，调用 `HAL_TIM_GenerateEvent` 强制立即更新。
- **状态**: ✅ 已解决。成功播放《超级玛丽》。
